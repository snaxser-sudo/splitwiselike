# Context Rule

- At the start of substantial work, read `AGENTS.md`, `README.md`, and
  `.codex/SNAPSHOT.md`.
- Check `git status --short` before editing so user changes are not overwritten.
- Use `rg` and `rg --files` for discovery.
- Keep working notes short and specific.
- If context is getting stale, re-read the relevant files instead of guessing.
- Update `.codex/SNAPSHOT.md` only when the user asks for a snapshot update or
  approves a proposed update.
