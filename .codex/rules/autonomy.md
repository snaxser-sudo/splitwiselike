# Autonomy Rule

- First respect the approved scope from the user.
- If the request is small and unambiguous, make the targeted change directly.
- For non-trivial work, propose the change and wait for approval before editing.
- Ask only when the next step is genuinely ambiguous, risky, or cannot be
  inferred from the repository.
- Prefer existing project patterns over new abstractions.
- Before adding a new function, endpoint, file, or class, search for an existing
  equivalent with `rg`.
