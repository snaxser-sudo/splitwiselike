# Commits Rule

- Do not commit, push, or create pull requests without explicit approval.
- Before any commit request, run `git status --short` and review changed files.
- Keep commit messages concise and project-focused.
- Never include AI, assistant, Claude, or code-generation attribution.
- Never use destructive git commands to undo edits. Revert your own changes with
  targeted patches only.
- Do not include secrets or local Supabase credentials in commits.
